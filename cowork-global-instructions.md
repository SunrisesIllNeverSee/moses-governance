# MO§E§™ Document Governance — Cowork Global Instructions

## Document Numbering (Always Active)

Every file created in any session gets a sequential 3-digit prefix: 001_, 002_, 003_. No exceptions.

### Naming: `[SEQ]_[descriptive-name].[ext]`

### Rules
- Three-digit prefix, incrementing by 1 for every new file
- Cross-references: when a new doc references an earlier doc, use the parent's number + alphabetical suffix (001a_, 001b_)
- Cross-reference docs do NOT consume the next sequential number — they branch from the parent
- Versions: when directly revising the same doc, append v2, v3 after the name before the extension
- Never skip or reuse numbers

### Document Header (top of every doc)
```
──────────────────────────────────────
DOC [SEQ] | [TOPIC/THEME TAG]
[YYYY-MM-DD HH:MM] | Session: [brief description]
──────────────────────────────────────
```

Cross-references add: `Refs: DOC [parent]`
Pivots add: `Pivot: [from → to]`

### Session Header (DOC 001)
First doc in every session. Includes:
- Operator: Luthen / Ello Cello LLC
- Timestamp
- Theme/topic
- Table of Contents (table format: Doc | Name | Topic | Time)
- TOC gets UPDATED as new docs are created, not recreated from scratch

### After Action Review (Final Doc)
Last doc in a session or at major pivots. Includes:
- What was accomplished
- Complete document index (full TOC table)
- Pivots (where direction changed and why)
- What carries forward (open items, next steps)
- Key decisions made during session

### Pivot Tracking
When conversation topic shifts significantly, the next doc's header includes a Pivot tag. The AAR captures all pivots.
